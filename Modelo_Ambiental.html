II. MODELO AMBIENTAL DEL SISTEMA

Modelo Ambiental

1. Declaración de Propósitos 2. Diagrama de Contexto 3. Lista de Acontecimientos (L.A)

1. Declaración de Propósitos del Sistema

Propósito: El nuevo Sistema de Gestión Académica y Administrativa del Instituto Simón Bolívar (ITSB) debe ser diseñado para centralizar, automatizar y optimizar todos los procesos críticos de la institución, desde la inscripción y el registro de notas hasta la gestión de personal y la emisión de reportes, garantizando la adaptabilidad a los constantes cambios normativos del sector educativo boliviano.

Debe proveer una plataforma integrada que:

  • Gestione de forma robusta y centralizada la información académica y de pagos de los estudiantes de las cinco sedes.
  • Permita a los docentes cargar calificaciones y registrar asistencia de forma segura y validada.
  • Facilite a la Dirección Académica y Administrativa la generación de reportes dinámicos (inscripciones, deserciones, pagos, historiales) para la toma de decisiones.
  • Integre completamente la gestión de recursos humanos del personal docente y administrativo.
  • Ofrezca interfaces web diferenciadas y seguras para Administradores, Docentes y Estudiantes.

2. Diagrama de Contexto (DC)

El Diagrama de Contexto define el límite del sistema, identificando el sistema central a ser desarrollado (Sistema de Gestión ITSB) y sus interacciones con las entidades externas.

Entidades Externas Clave:

  • Estudiante: Usuario que interactúa para inscripción, consulta de notas/pagos, y encuestas.
  • Docente: Usuario que interactúa para el registro de calificaciones, asistencia y consulta de carga horaria.
  • Administrativo (Secretaría/Contabilidad): Personal encargado de validar pagos, gestionar inscripciones, emitir facturas y generar reportes.
  • Rectoría/Dirección: Instancia superior que solicita reportes de rendimiento y toma decisiones estratégicas.
  • Facturador Electrónico: Sistema externo para el proceso de facturación y firma electrónica.

3. Lista de Acontecimientos (L.A)

Los siguientes son los eventos que el sistema debe reconocer y a los que debe responder, basados en los procesos críticos del ITSB:

Acontecimientos de Flujo de Datos (Input/Solicitud):

  • El Estudiante solicita inscripción: El sistema debe registrar la solicitud, verificar requisitos y generar la orden de pago.
  • El Estudiante realiza un pago: El sistema recibe la confirmación de pago y emite la factura electrónica.
  • El Docente registra las calificaciones parciales: El sistema debe recibir y validar las notas según los parámetros académicos.
  • El Docente registra la asistencia: El sistema debe recibir la lista de asistencia y actualizar el registro del estudiante.
  • El Administrativo ingresa la carga horaria: El sistema debe registrar las asignaturas, horarios y distribución docente.

Acontecimientos Temporales (Basados en el Tiempo):

  • Fin de Semestre: El sistema debe generar automáticamente las actas de calificaciones finales y los reportes de promedios.
  • Inicio del Período de Inscripción: El sistema debe activar la funcionalidad de inscripción masiva.
  • Mensualmente/Bimestralmente: El sistema debe generar reportes de morosidad, deserción y rendimiento para la Dirección.

Acontecimientos de Control (Estado/Cambio):

  • Se detecta una irregularidad en el registro de notas: El sistema debe notificar al supervisor académico.
  • Una normativa educativa es modificada: El sistema debe permitir al administrador modificar los parámetros de cálculo de notas/promedios de forma flexible.
  • El Estudiante finaliza su plan de estudios: El sistema debe iniciar el proceso de titulación (no completamente integrado, pero el evento debe ser gestionado).
